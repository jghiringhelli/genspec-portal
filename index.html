🔁 The Quality Gates Flywheel

Every project you build teaches you something that protects every project after it. When you discover a gate that would have caught a bug — you can contribute it back, and it becomes part of the registry for everyone.

1
Discover You find a class of bug that a specific check would catch. You add it to .forgecraft/project-gates.yaml with generalizable: true.
2
Contribute Run forgecraft contribute-gate. A GitHub issue is created with your gate definition and the real-world evidence of the bug it catches.
3
Review Community reviews the gate. Evidence + method-agnostic definition required. No vendor preference. If it catches real bugs, it ships.
4
Compound Merged gate ships to all users via registry. Contributors earn Pro credits. The methodology improves for everyone simultaneously.
